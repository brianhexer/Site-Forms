🚀 What is Site-Forms?

Site-Forms is a free, open-source form management system designed for GitHub Pages. It allows you to:

  • ✅ Create unlimited contact, newsletter, feedback, and survey forms
  • ✅ Send form responses directly to your email
  • ✅ Deploy on GitHub Pages at no cost
  • ✅ Use no backend servers or complex setup
  • ✅ Collaborate with team members using standard form templates
  • ✅ Share reusable form components across projects
  • ✅ Protect against spam and abuse automatically

✨ Key Features

📧 Email Delivery

Forms send emails instantly to your inbox using EmailJS, SendGrid, Firebase, or custom webhooks.

📱 Mobile Ready

All forms are fully responsive and work perfectly on phones, tablets, and desktops.

🔐 Secure

Built-in spam detection, rate limiting, input validation, and CORS protection.

♻️ Reusable

Use the same form model across multiple pages and projects. Share templates with your team.

⚡ No Backend

Everything works on the frontend. No servers, databases, or complex infrastructure needed.

📚 Well Documented

Complete guides, API reference, examples, and code documentation included.

⚡ Quick Start (5 Minutes)

1Sign Up for EmailJS

Go to emailjs.com and create a free account (200 emails/month included).

2Create Email Service

In EmailJS dashboard:

  • Click "Email Services"
  • Click "Add Service"
  • Select your email provider (Gmail, Outlook, etc.)
  • Follow the setup wizard

3Create Email Template

Create a template for your forms:

Name: contact_form Subject: New submission from {{from_name}} Content: From: {{from_name}} ({{from_email}}) Message: {{message}}

4Get Your Credentials

Copy these three values from EmailJS:

  • Service ID: service_xxxxx
  • Template ID: template_xxxxx
  • Public Key: your_public_key

5Test a Form

Visit one of the example forms and test it:

📬 Contact Form 📰 Newsletter ⭐ Feedback
✅ That's it! You now have a working form system. Fill out a form to receive an email!

🎨 Create Your Own Form

Step 1: Copy the Template

<form id="my-form" class="site-form" data-form-name="my-form" data-recipient="your-email@example.com" data-success-message="Thank you!" data-error-message="Please try again"> <div class="form-field"> <label for="name">Name</label> <input type="text" id="name" name="name" required> </div> <button type="submit" class="btn btn-submit">Send</button> </form> <script src="../js/form-model.js"></script> <script src="../js/form-handler.js"></script>

Step 2: Add Your Fields

<div class="form-field"> <label for="email">Email</label> <input type="email" id="email" name="email" required> </div> <div class="form-field"> <label for="message">Message</label> <textarea id="message" name="message" required></textarea> </div>

Step 3: Customize Data Attributes

Important attributes on the form tag:

  • data-recipient - Email to receive submissions
  • data-success-message - Message shown on success
  • data-error-message - Message shown on error
  • data-redirect-url - Optional redirect after submit

Step 4: Test & Deploy

  1. Test locally with Python: python -m http.server 8000
  2. Verify form submits and you receive email
  3. Push to GitHub: git add . && git commit -m "Add my form" && git push
  4. Auto-deployed to GitHub Pages! 🚀

❓ Frequently Asked Questions

Q: Is Site-Forms free?

Yes! The system itself is free and open-source. Email providers have free tiers (EmailJS: 200/month, SendGrid: 250/day).

Q: Do I need a backend server?

No! With EmailJS, everything runs in the browser. No backend needed.

Q: Can I customize the forms?

Yes! All forms are fully customizable. Edit HTML, CSS, and form fields as needed.

Q: How do I prevent spam?

Site-Forms has built-in spam detection, rate limiting (5 submissions/hour by default), and input validation.

Q: Can multiple people use this?

Yes! Each team member can create their own forms. The system uses a standard form schema so everything is compatible.

Q: What if I need more emails per month?

Upgrade your email provider plan, or use multiple providers (SendGrid, Firebase, etc.).

Q: How do I test forms locally?

Run: python -m http.server 8000 then visit localhost:8000/docs/

Q: Can I use this on a custom domain?

Yes! Just set up a custom domain in GitHub Pages settings and point your DNS to GitHub.
